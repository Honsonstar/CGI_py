# 多模态癌症生存预测实验报告

**生成时间**: 2026-01-23  
**实验环境**: conda环境 `causal`  
**数据集**: TCGA多癌症类型 (5折交叉验证)

---

## 1. 实验概述

### 1.1 实验目的
对比不同模态组合在癌症生存预测任务上的表现：
- **仅文本模式** (ab_model=1): 仅使用BioBERT编码的临床报告
- **仅基因模式** (ab_model=2): 仅使用基因表达数据
- **基因+文本模式** (ab_model=3): 多模态融合

### 1.2 对比基线
- **CFdemo**: 原始项目（包含文本+基因）
- **CFdemo_gene**: 仅基因版本

### 1.3 评估指标
- **C-index**: 一致性指数（主要指标，越高越好）
- **C-index IPCW**: 逆概率删失加权一致性指数
- **IBS**: 综合Brier得分（越低越好）

## 4. 结论与建议

### 4.1 主要结论

1. **多模态融合有效**: 基因+文本模式在BRCA、BLCA、HNSC上优于单模态
2. **文本信息价值高**: 
   - COADREAD上仅文本模式取得最佳结果(0.786)
   - 说明临床报告对某些癌症预测价值很高
3. **CFdemo整体最优**: 原始CFdemo在BRCA、STAD上表现最好
4. **癌症特异性明显**: 不同癌症适合不同模态组合

### 4.2 改进建议

1. **修复BLCA仅文本模式**: 需要调查KM图生成时数据为空的原因
2. **针对癌症类型选择模型**: 
   - COADREAD: 优先使用文本模态
   - BRCA/STAD: 使用CFdemo
   - HNSC/BLCA: 使用基因+文本融合
3. **进一步优化多模态融合策略**: 当前简单拼接可能不是最优方案

## 5. 技术细节

### 5.1 训练配置
```
epochs: 20
batch_size: 1
learning_rate: 0.0005
optimizer: RAdam
weight_decay: 0.0001
alpha_surv: 0.5
multitask_weight: 0.12
splits: 5-fold cross-validation (random)
```

### 5.2 模型架构
- **文本编码器**: BioBERT (768维)
- **基因编码器**: SNN (256维)
- **融合方式**: 拼接 (基因256 + 文本768 = 1024维)

### 5.3 BLCA仅文本训练失败原因
```
错误信息: ValueError: Empty array/Series passed in.
位置: lifelines KaplanMeierFitter.fit()
原因: 风险分组后某一组为空，导致KM图无法生成
```
